# BUG-003 — total_pages calculado com divisão inteira (floor) em vez de ceil

**Severidade:** Média
**Endpoint:** GET /api/v1/services
**Código-fonte:** api/main.go, linhas 162 e 187
**Status:** Reproduzível e coberto por teste automatizado

## O que acontece

O campo total_pages na resposta paginada é calculado como total / per_page usando divisão inteira de Go, que trunca o resultado (floor), em vez de arredondar para cima (ceil).

Quando há 11 itens e per_page=10, o cálculo é 11/10 = 1, mas a página 2 existe e contém 1 item (s011 — Bolsa Família).

Comentários no código:
- "total_pages uses integer division (floor) instead of ceiling"
- "should use math.Ceil or integer ceiling arithmetic"

## O que deveria acontecer

total_pages deve refletir o número real de páginas disponíveis. Para 11 itens com per_page=10, deveria retornar:

    {"total":11, "page":1, "per_page":10, "total_pages":2}

## Como reproduzir

Página 1 (mostra total_pages:1 — errado):

    curl -i -H "Authorization: Bearer qa-challenge-token" \
      "http://localhost:8080/api/v1/services"

Resposta: "total":11, "page":1, "per_page":10, "total_pages":1

Página 2 (retorna dados reais, contradizendo total_pages):

    curl -i -H "Authorization: Bearer qa-challenge-token" \
      "http://localhost:8080/api/v1/services?page=2"

Resposta: retorna s011 com sucesso, mas mantém total_pages:1. Ou seja: a página 2 existe, mas a API diz que não existe.

## Impacto

- Dados invisíveis ao usuário: front-ends que usam total_pages para renderizar controles de paginação (botões "próxima", numerados) vão esconder a última página. No caso atual, o serviço s011 — Bolsa Família fica invisível para o cidadão se o front-end seguir o contrato.
- Contrato quebrado: consumidor não pode confiar em total_pages como fonte da verdade — precisa calcular do lado dele, contornando o que a API deveria entregar pronto.
- Regressão silenciosa: como a contagem está errada mas os dados vêm, bugs relacionados passam fácil em testes superficiais.

## Hipótese de correção

Go não tem operador de divisão com ceil nativo para inteiros. Pode-se usar aritmética:

    totalPages := (total + perPage - 1) / perPage

Ou: int(math.Ceil(float64(total) / float64(perPage)))
